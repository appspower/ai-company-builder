## 1. 제품 범위 (V1 — Phase 1: Frontend)

Phase 1은 **더미 데이터 기반 프론트엔드**입니다. 백엔드 API 연동 없이 모든 화면이 하드코딩된 JSON 데이터로 렌더링됩니다. 목적은 **사용자 경험(UX)과 업무 흐름을 시각적으로 검증**하는 것입니다.

### 구현 범위

| 포함 (Phase 1 완료) | 미포함 (Phase 2 이후) |
|---|---|
| 7개 화면 UI 렌더링 | 백엔드 API 연동 |
| 더미 데이터 기반 인터랙션 | 실제 AI 모델 호출 |
| 칸반 드래그앤드롭 + 5가지 Group By | 실시간 WebSocket |
| 3단계 결재 (자동/최종/단계별) | 워크플로우 엔진 (Inngest) |
| 인라인 스플릿 뷰 (에이전트/미션/산출물/활동) | 파일 업로드/다운로드 |
| 퀵 필터 칩 + 검색 + 정렬 | 인증/로그인 |
| 에이전트 업무 지침 (마크다운) + 참고 자료 + 도구 | RAG 벡터 검색 |
| 산출물 즐겨찾기 + 마크다운 미리보기 | 실제 파일 스토리지 |
| 활동 로그 (실행 기록 + 이벤트 피드) | 실시간 에이전트 모니터링 |
| 다크/라이트 모드 | 멀티유저 인증 |

## 2. 정보 구조 (Information Architecture)

### 2.1 화면 계층

```
Level 0: 워크스페이스 로비
         └── 회사 카드 클릭 시 Level 1 진입

Level 1: 회사 내부 (사이드바 네비게이션, 병렬 형제 관계)
         ├── 홈 (대시보드)
         ├── 조직 & 에이전트
         ├── 미션 센터
         ├── 활동 로그
         ├── 산출물 보관함
         └── 설정 (하단 분리)

Level 2: 인라인 상세 뷰 (오른쪽 스플릿 패널, 딤 없음)
         ├── 에이전트 상세 (4탭: 기본정보/업무지침/도구/참고자료)
         ├── 미션 상세 (상태/타임라인/산출물/결재)
         ├── 산출물 상세 (미리보기/메타/버전/연결미션)
         └── 실행 기록 상세 (통계/시간바/엔티티링크)
```

### 2.2 URL 구조

```
/                                    → 워크스페이스 로비
/:workspaceId                        → 홈 대시보드
/:workspaceId/agents                 → 조직 & 에이전트
/:workspaceId/missions               → 미션 센터
/:workspaceId/activity               → 활동 로그
/:workspaceId/artifacts              → 산출물 보관함
/:workspaceId/settings               → 설정
```

### 2.3 네비게이션 원칙

| 요소 | 패턴 |
|---|---|
| 워크스페이스 전환 | 좌상단 드롭다운 |
| 회사 내 이동 | 왼쪽 사이드바 (5개 주 메뉴 + 설정 하단) |
| 상세 정보 | 인라인 스플릿 패널 (읽기), Sheet 모달 (편집/결재) |
| 화면 간 연결 | 모든 엔티티가 상호 링크 (미션↔에이전트↔산출물↔활동) |

### 2.4 사이드바 구성

```
[🎬 AI 숏폼 팩토리 ▼]      ← 워크스페이스 전환
──────────────────
🏠 홈
👥 조직 & 에이전트
🚀 미션 센터          [2]   ← 결재 대기 건수
📋 활동 로그          [3]   ← 실행 중 건수
📦 산출물 보관함
──────────────────
⚙️ 설정                     ← 하단 분리
🌓 테마 전환
← 로비로 돌아가기
```

## 3. 화면별 행동 명세

### 3.1 워크스페이스 로비 (Level 0)

**사용자 목표:** 운영 중인 회사 목록을 보고 입장하거나 새 회사를 만든다.

- 회사 카드 그리드: 아이콘, 이름, 설명, 요약 수치(에이전트수/미션수/승인대기), 마지막 활동 시간
- "+ 새 회사 만들기" 카드 → 생성 모달 (아이콘 선택 + 이름 + 설명)
- 카드 클릭 → 해당 회사 홈 대시보드로 이동
- 빈 상태: "첫 번째 AI 회사를 만들어 보세요" CTA

### 3.2 홈 대시보드 (Level 1 — 기본 화면)

**사용자 목표:** 회사 전체 현황 파악 + 긴급한 승인 건 바로 대응.

**위젯:**
- **결재함** (최상단): 승인 대기 미션 + 인라인 승인/반려 버튼
- **미션 현황**: 5개 상태별 숫자 (클릭 → 미션 센터 이동)
- **에이전트 현황**: 4개 상태별 숫자 + 최근 활동 (클릭 → 조직 화면 이동)
- **최근 산출물**: 최근 5건 (클릭 → 산출물 화면 이동)

### 3.3 조직 & 에이전트 (Level 1)

**사용자 목표:** 부서와 AI 직원을 배치하고, 각 직원의 역할/능력을 설정한다.

**레이아웃:** 부서 사이드 | 에이전트 카드 그리드 | 인라인 상세 패널

**에이전트 카드:**
- 상태등 + 이름 + 역할 + LLM 모델 뱃지 + 마지막 활동
- hover 시 컨텍스트 메뉴(⋯): 편집, 복제, 삭제
- 비활성 에이전트 50% 투명

**에이전트 상세 (인라인 스플릿, 4탭):**
- **기본 정보**: 부서/엔진/상태/활동 + 현재 진행 작업(파란 박스+스피너) + 최근 미션
- **업무 지침**: 마크다운 에디터 (템플릿: 성격&톤, 업무프로세스, 규칙&제약, 출력형식)
- **도구**: 할당된 스킬 목록 (아이콘+이름+설명)
- **참고 자료**: 파일 리스트 (유형아이콘, 크기, 상태뱃지) + 업로드 영역

**액션 바:** 활성/비활성 토글 + 테스트 대화 버튼(Phase 2)
**편집:** "편집" 클릭 → Sheet 모달로 전환 (이름/역할/부서/페르소나/LLM 편집)
**부서 관리:** 좌측 부서 리스트 + 인라인 추가 폼

### 3.4 미션 센터 (Level 1)

**사용자 목표:** 업무를 칸반으로 관리하고, 결재하고, 다양한 관점으로 본다.

**필터 바:**
- 퀵 필터 칩 5개: ⚡긴급, 🕐결재대기, ⚠️마감임박, 📋진행중, ✅완료제외
- 부서 드롭다운 + 검색

**Group By 버튼 5개:** 상태별 / 우선순위별 / 부서별 / 담당직원별 / 결재방식별
**뷰 토글:** Board ↔ List

**칸반 보드 (Board 뷰):**
- 5개 컬럼: 대기 → 진행 중 → 검토 중 → 결재 대기 → 완료
- 반응형: 사이드 패널 열리면 컬럼 자동 축소
- 드래그앤드롭 (상태별 그룹에서만)

**미션 카드:**
- 우선순위 도트(🔴긴급/🟠높음/🔵보통/⚪낮음) + 부서 뱃지
- 제목, 진행률 바(%), 생성일, 마감일(초과 시 ⚠️), 산출물 수, 결재방식 뱃지

**미션 상세 (인라인 스플릿):**
- 제목 + 상태/우선순위/결재방식 뱃지
- 반려 사유 배너 (빨간, 있을 때만)
- 완료 요약 카드 (초록, 완료 시 — 단계수/산출물/완료일)
- 설명, 메타(부서/생성일/마감일/산출물), 담당 직원(클릭 가능), 타임라인, 산출물(클릭 가능)
- "결재 처리하기" 버튼 → Sheet 모달(승인/반려+피드백)

**3단계 결재:**
- 🟢 자동 완료: 에이전트가 끝까지 진행, 결과만 CEO 확인
- 🟡 최종 결재: 마지막 단계에서만 CEO 승인
- 🔴 단계별 결재: 각 단계마다 CEO 승인 필요

**미션 생성 (Sheet):**
- 제목, 설명, 부서+우선순위(2열), 마감일, 결재 방식(3카드 선택)

### 3.5 활동 로그 (Level 1)

**사용자 목표:** AI 직원들이 지금 뭘 하는지, 무슨 일이 있었는지 모니터링한다.

**2개 탭:** 실행 기록 | 이벤트 피드

**실행 기록 탭:**
- 요약 카드 4개: 실행 중 / 전체 실행 / 총 토큰 / 총 비용
- 퀵 필터 칩 4개: 🔄실행중, ❌실패, 🕐오늘, 💰고비용
- Group By 4가지: 전체 / 직원별 / 미션별 / 상태별
- 검색 (미션/직원/작업 이름)
- 실행 행: 상태아이콘(스피너), 작업명, 미션명, 에이전트, LLM뱃지, 토큰, 비용, duration 바
- 클릭 → 인라인 상세 (통계 그리드 + 시간 바 + 미션/에이전트 링크)

**이벤트 피드 탭:**
- 카테고리 필터 + 검색
- 이벤트 카드: 이모지 + 제목 + 설명 + 시간 + 유형 라벨 + 좌측 컬러 바
- 클릭 → 관련 화면 이동 ("미션 센터에서 보기 →")

### 3.6 산출물 보관함 (Level 1)

**사용자 목표:** AI 직원이 만든 결과물을 검색, 필터, 미리보고, 관리한다.

**필터 바:**
- 퀵 필터 칩 4개: ⭐즐겨찾기, 🕐승인대기, ⚡최근생성, ✅게시완료
- 유형 드롭다운 + 미션 드롭다운 + 검색
- 뷰 토글: List ↔ Grid

**Group By 버튼 4개:** 전체 / 미션별 / 직원별 / 유형별
**컬럼 정렬:** 파일명/버전/상태/생성일 (헤더 클릭)

**리스트 뷰 컬럼:** ⭐ | 파일명(아이콘) | 직원 | 연결 미션 | 버전 | 상태(컬러닷) | 생성일
**즐겨찾기:** 별표 토글 + "즐겨찾기" 퀵필터

**산출물 상세 (인라인 스플릿):**
- 파일아이콘 + 이름 + 별표 + 닫기
- 상태/버전/LLM뱃지
- **미리보기**: 마크다운 콘텐츠 렌더링 (document 타입) / 플레이스홀더 (기타 타입)
- 메타: 직원, 생성일, 부가정보
- 연결 미션 (클릭 → 미션 센터)
- 버전 이력 (v1→v2→v3 타임라인, 현재 강조)
- 다운로드 버튼

### 3.7 설정 (Level 1)

**6개 탭:**

| 탭 | 내용 |
|---|---|
| **회사 정보** | 기본정보(아이콘/이름/설명) + 사업자정보(구분/등록번호/대표자/업태/업종/주소) + 담당자(이름/이메일/연락처) |
| **AI 엔진** | 기본 모델 선택(드롭다운) + API 키 관리(Claude/OpenAI/Gemini, 비밀번호 토글, 연결 상태) |
| **멤버 관리** | 오너👑/관리자🛡️/멤버👤 역할 목록 + 초대 버튼(Phase 2) + 역할 안내 |
| **사용량** | 월간 요약(토큰/비용/실행/산출물) + 모델별 사용량 바 + 일별 테이블 |
| **알림** | 4개 토글(결재요청/미션완료/에이전트에러/산출물생성) |
| **데이터 관리** | 위험 영역(초기화/삭제) — Phase 2 |

## 4. 공통 UI 패턴

### 4.1 인라인 스플릿 뷰 (모든 상세)

- 오른쪽 400px 패널, 왼쪽은 선명하게 유지 (딤 없음)
- 다른 항목 클릭 시 패널 내용만 교체
- X 버튼으로 닫기
- 편집/결재 시에만 Sheet 모달 전환

### 4.2 퀵 필터 칩

- 가로 배열, 토글식 (ON/OFF), 여러 칩 동시 활성 (AND 로직)
- 미션/산출물/활동 3개 화면에서 동일 패턴

### 4.3 Group By 버튼

- 배경 있는 토글 바 (bg-muted/50), 활성 버튼에 shadow
- 미션/산출물/활동 3개 화면에서 동일 패턴

### 4.4 빈 상태 패턴

- text-5xl 이모지 + text-lg 제목 + text-sm 설명 + CTA 버튼

### 4.5 토스트 알림

- Sonner, 우측 하단, 성공(초록)/에러(빨강)/정보(파랑)

## 5. 사용자 시나리오 (검증 완료)

### 시나리오 A: 회사 생성 + 조직 세팅
로비 → 회사 생성 → 입장(홈) → 조직&에이전트 → 부서 추가 → 에이전트 추가(이름/역할/업무지침/AI엔진)

### 시나리오 B: 미션 실행 + 승인
홈 → 미션 센터 → 새 미션(제목/부서/우선순위/마감일/결재방식) → 칸반 진행 → CEO 승인 또는 반려(피드백)

### 시나리오 C: 산출물 확인 + 활용
홈 → 산출물 보관함 → 필터/검색 → 인라인 상세(미리보기) → 연결 미션 → 미션 상세

## 6. 더미 데이터 명세

### 6.1 워크스페이스 (4개)
미국/독일 구매대행 🌍 | 쿠팡 로켓그로스 🚀 | AI 숏폼 팩토리 🎬 | 시니어 타겟 SNS 💜

### 6.2 에이전트 (23명, 16개 부서)
- 각 에이전트: 이름, 역할, 업무 지침(마크다운), LLM 모델, 참고 자료(파일 리스트), 도구/스킬, 활성/비활성, 상태(활동/대기/경고/에러)

### 6.3 미션 (13건, ws-3 중심)
- 각 미션: 제목, 설명, 부서, 우선순위(4단계), 마감일, 결재방식(3단계), 진행률, 단계 타임라인, 산출물 수, 반려 피드백

### 6.4 산출물 (10건, 전 워크스페이스)
- 각 산출물: 파일명, 유형, 버전, 상태, 즐겨찾기, 마크다운 콘텐츠(5건), 메타(LLM/토큰/크기)

### 6.5 활동 로그
- 실행 기록 13건: 상태(running/success/failed), 에이전트, 미션, 토큰, 비용, 소요시간
- 이벤트 15건: 미션 생성/시작/완료, 결재 요청, 산출물 생성, 에이전트 상태 변경
